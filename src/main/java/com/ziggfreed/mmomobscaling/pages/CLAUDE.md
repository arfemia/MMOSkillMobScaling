# pages/ - in-game admin config UI

Router for `com.ziggfreed.mmomobscaling.pages`. The mod's ONLY interactive UI page (the HUD overlays live in `hud/`). Opened by `/mobscaling ui` (admin-gated at the command); do not add another entry point.

- **[`MobScalingAdminPage`](MobScalingAdminPage.java)** extends the engine `InteractiveCustomUIPage<EventData>` (raw, not the toast base - it uses an inline `#MmoscalingStatus` label for save feedback). Four tabs over the shared `ziggfreed-common` `@ZigDecoratedFrame`: **Global** (Enabled, ActivePreset, Intensity, RaritySpawnChance, PlayerScaling, Difficulty caps, DistanceEscalation), **Zone HUD** + **Inspector HUD** (enable / position preset + offsets / sub-toggles / range), and **Worlds** (a per-world `WorldOverrides` CRUD list + add/edit editor). `.ui`: `Common/UI/Custom/Pages/MmoscalingAdminPage.ui`; per-world rows append the common `Pages/ZigListRow.ui`.

## Hard rules (do NOT regress)

- **Every edit persists through ONE path: `config/MobScalingOwnerWriter`** (-> the common `JsonOverrideWriter` -> the owner file `mods/MmoMobScaling/mob-scaling.json` -> `MobScalingConfig.refreshFromDisk`). The page NEVER writes the file itself and NEVER mutates `MobScalingConfig` fields directly. HUD + preset edits ALSO live-apply (`ZoneDifficultyHud`/`MobInspectorHud.setEnabledForAllOnline`/`refreshPositionForAllOnline`, `swapActivePreset`); spawn-path knobs (intensity, caps, escalation) need no push (read per spawn). The SAME `MobScalingOwnerWriter` path backs the `/mobscaling` command, so the two never diverge.
- **`Enabled` needs a restart** (the zero-cost registration gate registers systems only at `setup()`): persist + flip the field but the UI shows the "takes effect on restart" note. Never try to live-(un)register.
- **RICH buttons only.** Every labelled button is a `Button` + inner `#Label` (`@ZigRichRowBtnStyle` / `@ZigRichPrimaryBtnStyle`), label pushed via `ZigRichButton.text` / `SettingsUiUtil.setToggle` / `setTabActive` on `.TextSpans` - NEVER a `TextButton.Text`. All display text is a client-resolved `Message` on `.TextSpans`; only DATA values (a Match pattern, a number, a preset id) are literal on `.Text`.
- **Per-world editor seeds from the FOLDED effective override** (`cfg.effectiveWorldOverride(match)`) and Save writes the WHOLE entry (`upsertWorldOverride`), because the fold replaces a `WorldOverride` by `Match` wholesale - an omitted leaf would be lost. Blank editor fields = inherit (leaf omitted); the tri-state dropdowns carry an Inherit option. StatCurve per-world stays file-authored.
- **Namespace every id `#Mmoscaling*`** (the client UI namespace is flat + cross-mod). CRUD-row bare ids (`#Title`/`#EditBtn`) are safe only because the indexed parent (`#MmoscalingWorldList[i]`) scopes them.
- **The shared settings-UI toolkit is `ziggfreed-common`** (`ui/SettingsUiUtil`, `ui/ZigRichButton`, `ui/hud/HudPosition`, `util/JsonOverrideWriter`, `Pages/ZigListRow.ui`) - reuse it, do not re-derive a binding helper / row here.
